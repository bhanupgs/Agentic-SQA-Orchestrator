## Agent 3: SDET Automation Agent
### Role & Objective

The SDET Automation Agent is the "builder" of the ecosystem. 
It takes the Strategy Map generated by Agent 2 and produces high-fidelity, production-ready automation scripts. 
By utilizing Retrieval-Augmented Generation (RAG), this agent ensures that all generated code adheres to internal library standards, security protocols, and established CI/CD patterns.

This reduces manual scripting overhead and ensures that the automation suite remains scalable and maintainable without human intervention in the repetitive coding phase.

### Core Responsibilities

**High-Fidelity Code Generation:** Produces executable Python scripts using frameworks like Pytest, Robot Framework, or Cypress.
**Standard Alignment (RAG):** References a vector database of "Gold Standard" internal scripts to ensure naming conventions, logging, and error-handling patterns are followed.
**Protocol-Specific Scripting:** Generates specialized validation logic for Access APPs, L2/L3/L4/L7 protocols, IPsec tunnels, and OSPF configurations.
**Sanity & Linting:** Performs an initial self-check of the code for syntax errors or library incompatibilities before handing it off to the execution layer.
**CI/CD Integration:** Formats output for immediate ingestion into pipelines like GitHub Actions or Jenkins.

### Technical Implementation
**Intelligence Layer:** LLM-driven agent optimized for code generation (e.g., CodeLlama, GPT-4o, or Gemini Pro).
**RAG Architecture:** A Pinecone or Weaviate vector store containing your team's proprietary test libraries and common utility functions.
**Validation Loop:** Integrated with a local linter (like Flake8 or Pylint) to ensure the code is clean before it leaves the agent's environment.

### Code Generation Example
The agent transforms the "Strategy Map" into a functional test file:

Python
import pytest
from internal_net_lib import TunnelManager

# Generated based on STRAT-IPSEC-STABILITY requirement
@pytest.mark.resilience
def test_ipsec_tunnel_failover():
    """
    Validates tunnel re-establishment within 30s as per Agent 1 requirement.
    """
    tunnel = TunnelManager(id="GW-V-01")
    tunnel.trigger_failover()
    
    assert tunnel.wait_for_state("ESTABLISHED", timeout=30)
    assert tunnel.get_packet_loss() == 0

### Next Steps
**Execution Layer:** The generated code is pushed to Antigravity or a similar sandboxed environment for execution.


**Failure Analysis:** If the test fails, the results are sent back to the Orchestration Supervisor for a Root Cause Analysis (RCA) review.
